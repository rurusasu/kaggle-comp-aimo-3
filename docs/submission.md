# AI Mathematical Olympiad - Progress Prize 3 - Submission Format

## Submission Method

Submissions to this competition must be made through Kaggle Notebooks using the provided Python evaluation API.

The API serves test set instances one-by-one in random order for the public leaderboard, and uses fixed, random order for the private leaderboard.

To use the API, follow the template in this notebook: https://www.kaggle.com/code/ryanholbrook/aimo-3-submission-demo

## Notebook Requirements

- CPU Notebook <= 9 hours run-time
- GPU Notebook <= 5 hours run-time
- Internet access disabled
- Freely & publicly available external data is allowed, including pre-trained models
- Submission file must be generated by the API.

## Answer Format

Every ground-truth label is an integer between 0 and 99999, inclusive. Any modulo calculation required is explicitly stated in the problem statement, meaning all problems have answers in this range without any further adjustments.

## Private Evaluation

After the submission deadline, submission notebooks will be run **twice** over the private test set and their predictions concatenated into a single submission file. The penalized accuracy scoring applies:

- Both runs correct: score = 1
- One run correct, one incorrect: score = 0.5
- Neither correct: score = 0

## Submission Limits

- Maximum of one (1) Submission per day.
- You may select up to one (1) Final Submission for judging.

## Runtime Notes

Submission runtimes have been obfuscated. If you repeat the exact same submission you will see up to 30 minutes of variance in the time before you receive your score.
